## üìã Essential Workflows

### Making Your First Contribution

```bash
# 1. Fork and clone
git clone https://github.com/your-username/crazy-gary.git
cd crazy-gary

# 2. Set up environment
npm install
./scripts/setup-dev.sh

# 3. Create feature branch
git checkout -b fix/typo-in-readme

# 4. Make changes and test
npm test
npm run lint

# 5. Commit using our standards
git commit -m "docs(readme): fix typo in installation section"

# 6. Push and create PR
git push origin fix/typo-in-readme
```

### Bug Report Process

1. **Search Existing Issues**: Check if bug is already reported
2. **Use Bug Template**: Follow [Issue Reporting Guidelines](./ISSUE_REPORTING_GUIDELINES.md)
3. **Provide Details**: Environment, steps to reproduce, expected vs actual behavior
4. **Include Screenshots**: Visual evidence when applicable

### Feature Request Process

1. **Check Roadmap**: Review existing feature requests
2. **Use Feature Template**: Follow [Issue Reporting Guidelines](./ISSUE_REPORTING_GUIDELINES.md)
3. **Provide Context**: User stories, acceptance criteria, technical considerations
4. **Engage Community**: Discuss in GitHub Discussions

### Code Review Process

1. **Review Guidelines**: Follow [Code Review Guidelines](./CODE_REVIEW_GUIDELINES.md)
2. **Check Requirements**: Ensure PR meets all criteria
3. **Provide Constructive Feedback**: Focus on helping, not criticizing
4. **Use Standard Responses**: Templates provided in guidelines
